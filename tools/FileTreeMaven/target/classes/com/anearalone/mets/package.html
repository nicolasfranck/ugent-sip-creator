WARNING: There are a handful of elements that are not supported. Unmarshaling and then marshaling a record that contains these elements will result in data loss. See the Javadoc for details.

Intends to provide a dependency-free API for creating and manipulating METS objects. From the METS Schema:

A METS document consists of seven possible subsidiary sections: metsHdr (METS document header), dmdSec (descriptive metadata section), amdSec (administrative metadata section), fileGrp (file inventory group), structLink (structural map linking), structMap (structural map) and behaviorSec (behaviors section).

A few notes...

  • Creating a new Mets object and writing it out with no further manipulation via MetsWriter will always result in a valid METS record. No more, no less. The only major implication of this is that the list of StructMaps (Metss#structMap) is prepopulated with a single empty Structmap instance.
  • There are no setter methods for repeated elements; they are represented as Lists, and are manipulated by getting the List and using its methods.
